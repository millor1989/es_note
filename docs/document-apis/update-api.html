Update Multi_Get

1、Update API

基于提供的脚本或 document 片段，更新指定的 document。从 index 获取 document，执行更新，index 更新后的 document。会使用版本来确保在 “get” 和 “reindex” 期间不会更新。

这个操作意味着 document 的 full reindex

例子：

PUT test/_doc/1
{
    "counter" : 1,
    "tags" : ["red"]
}

1.1、脚本更新

增加 counter 值：

POST test/_doc/1/_update
{
    "script" : {
        "source": "ctx._source.counter += params.count",
        "lang": "painless",
        "params" : {
            "count" : 4
        }
    }
}

往 tags 集合中增加 tag：

POST test/_doc/1/_update
{
    "script" : {
        "source": "ctx._source.tags.add(params.tag)",
        "lang": "painless",
        "params" : {
            "tag" : "blue"
        }
    }
}

1.2、使用 document 片段更新

POST test/_doc/1/_update
{
    "doc" : {
        "name" : "new_name"
    }
}

如果同时指定了 doc 和 script 则忽略 doc。

2、Update By Query API

_update_by_query 的最简单应用是对 index 的每个 document 执行更新而不改变 document 的 source。

POST twitter/_update_by_query?conflicts=proceed

与 _delete_by_query 类似，_update_by_query 也是根据 index 快照进行更新操作。也会有版本冲突的问题。

相比 Update API，增加了 query ：

POST twitter/_update_by_query
{
  "script": {
    "source": "ctx._source.likes++",
    "lang": "painless"
  },
  "query": {
    "term": {
      "user": "kimchy"
    }
  }
}

也支持 routing 参数。

通知指定一个 pipeline _update_by_query 还可以使用 Ingest Node 特性：

PUT _ingest/pipeline/set-foo
{
  "description" : "sets foo",
  "processors" : [ {
      "set" : {
        "field": "foo",
        "value": "bar"
      }
  } ]
}


POST twitter/_update_by_query?pipeline=set-foo

3、Multi Get API

Multi get API 返回多个 documents。响应包含一个 docs 的数组——包含了获取到的 documents。

GET /_mget
{
    "docs" : [
        {
            "_index" : "test",
            "_type" : "_doc",
            "_id" : "1"
        },
        {
            "_index" : "test",
            "_type" : "_doc",
            "_id" : "2"
        }
    ]
}

写法可以改为针对某个 index 的：

GET /test/_mget
{
    "docs" : [
        {
            "_type" : "_doc",
            "_id" : "1"
        },
        {
            "_type" : "_doc",
            "_id" : "2"
        }
    ]
}

或者针对 type 的：

GET /test/_doc/_mget
{
    "docs" : [
        {
            "_id" : "1"
        },
        {
            "_id" : "2"
        }
    ]
}

也可以使用如下简化的写法：

GET /test/_doc/_mget
{
    "ids" : ["1", "2"]
}

支持 routing 参数。
